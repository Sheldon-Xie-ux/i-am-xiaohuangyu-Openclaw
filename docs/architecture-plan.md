## 推荐方案：双Agent架构

### 🐟 main (主控Agent)
- **职责**：日常助理、内容创作、开发、研究、社交、GitHub管理
- **模型**：Kimi K2.5
- **触发**：本群 + 其他非交易群

### 📊 trader (交易Agent)
- **职责**：币安/OKX量化交易、策略执行、风控监控
- **模型**：Kimi K2.5
- **触发**：仅交易专用群

## 分层记忆架构

| 层级 | 内容 | 存储位置 |
|------|------|----------|
| **即时** | 当前20-50轮对话 | 上下文窗口 |
| **短期** | 今日关键决策 | `memory/YYYY-MM-DD.md` |
| **长期** | 用户偏好、项目 | `MEMORY.md` |
| **档案** | 历史记录 | `memory/YYYY-MM-DD.md` |

---

## 上下文问题解决方案

### 为什么压缩不能彻底解决问题？

压缩 = 选择性**失忆**
- 长期必然丢失关键信息
- 无法区分"重要"和"冗余"
- 单Agent上限就是262k tokens

### 彻底的解决方案：分层记忆 + 多Agent

1. **每日自动归档**
   - 每天结束时，自动总结今日对话
   - 写入 memory/YYYY-MM-DD.md
   - 清空当前上下文

2. **会话启动时加载记忆**
   - 读取 MEMORY.md（长期记忆）
   - 读取今日 memory 文件（短期记忆）
   - 构建精简但完整的上下文

3. **关键信息标记**
   - 重要决策用 `[重要]` 标记
   - 自动提取到 MEMORY.md
