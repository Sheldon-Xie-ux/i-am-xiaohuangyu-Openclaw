# 记忆系统 V2 - 轻量级架构方案
# 更新时间: 2026-02-28
# 作者: 小黄鱼 (第九代)

---

## 📋 方案概述

基于第九代部署经验，设计的新记忆系统，核心目标：
- ✅ **防卡死**: 硬性限制 MEMORY.md ≤ 50 行
- ✅ **记得住**: 三级记忆架构（热/温/冷）
- ✅ **易维护**: 自动归档，按需检索

---

## 🎯 核心问题

| 问题 | 旧方案 | 新方案 |
|------|--------|--------|
| MEMORY.md 膨胀 | 无限制，持续增长 | 硬性 ≤50 行 |
| 群聊记忆隔离 | 群聊不加载个人记忆 | 专用 group-context 文件 |
| 信息碎片化 | 分散在多个 daily log | 统一归档到 archive-YYYY-MM.md |
| 启动 token 消耗 | 可能 5K+ | <500 token |

---

## 🏗️ 三级记忆架构

```
┌─────────────────────────────────────────┐
│  L1: 热记忆（当前会话）                   │  ← 8K token 内
│  - 实时上下文                            │
│  - 本轮对话关键信息                       │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│  L2: 温记忆（MEMORY.md）                 │  ← 硬性限制 50行
│  - 只保留：人物、本周大事、高频指令       │
│  - 超限时自动归档到冷记忆                 │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│  L3: 冷记忆（archive-YYYY-MM.md）        │  ← 完整历史
│  - 详细日志                             │
│  - 按需检索，不常驻加载                   │
└─────────────────────────────────────────┘
```

## 📁 文件结构

```
workspace/
├── MEMORY.md                      # 温记忆（≤50行，启动加载）
├── SOUL.md                        # 核心人格（不变）
├── USER.md                        # 用户信息（不变）
├── AGENTS.md                      # 工作区规则（不变）
└── memory/
    ├── archive-2026-02.md         # 冷记忆（详细历史）
    ├── group-trading-context.md   # 群聊专用上下文
    └── YYYY-MM-DD.md              # 当日详细日志（临时）
```

## 🔒 防膨胀机制

```python
当 MEMORY.md > 50行时:
    1. 提取超过30行的旧内容
    2. 迁移到 archive-YYYY-MM.md
    3. 保留最新30行 + 状态栏
    4. 添加"更多历史见 archive-XXX.md"链接
```

## 📊 Token 消耗对比

| 场景 | 旧方案 | 新方案 | 风险 |
|------|--------|--------|------|
| 启动加载记忆 | 可能 5K+ | ~200 | ✅ 极低 |
| 单次对话 | 1-3K | 1-3K | ✅ 可控 |
| 长期运行 | 持续增长 | 稳定在 <2K | ✅ 不会卡死 |

## 🚀 部署步骤

1. **备份旧记忆**
   ```bash
   mv MEMORY.md MEMORY-old.md
   ```

2. **创建新结构**
   ```bash
   mkdir -p memory
   touch memory/archive-$(date +%Y-%m).md
   touch memory/group-trading-context.md
   ```

3. **使用新模板**
   复制 [MEMORY-v2.md](./MEMORY-v2.md) 内容到 `MEMORY.md`

4. **迁移历史**
   把旧记忆中的详细内容移到 `archive-YYYY-MM.md`
